## Architecture summary

- **Entry points:** Rust+Tauri 2 core (`apps/desktop/src-tauri`, 35 `#[tauri::command]`s in `lib.rs`) owns all orchestration (auto-loop, slots, worktrees, dep-ordering, circuit breaker in `m2/`). Thin Bun NDJSON sidecar (`apps/sidecar`) wraps `packages/engine` (Claude Agent SDK). React 19 + Vite SPA (`apps/web`) in the webview.
- **Data flow (run a task):** UI `invoke('run_task')` → `task.rs` → `m2/coordinator.rs` leases slot → spawns sidecar child (`m2/provider.rs`, tokio) → writes NDJSON `SurfaceCommand` to stdin → sidecar `SessionManager.dispatch` → SDK loop → `NightcoreEvent` on stdout → Rust forwards verbatim inside `nc:session` → web `listen('nc:session')` → board re-renders. Permission prompts park in the engine and round-trip.
- **Boundaries:** contracts source-of-truth = `packages/contracts` (zod); persistence = `store.rs`/`settings.rs`/`transcript.rs` + `packages/storage`; process spawn = `m2/provider.rs`+`platform/mod.rs` and `packages/tools` (execFile).
- **Verified strong:** serde-additive discipline (every persisted struct has `#[serde(default)]`, legacy-load tests exist); process-spawn safety (arg-array, no shell interpolation); CSP + DOMPurify single-sink; slot/breaker/settings locks correct; PII/secret logging discipline clean.

## Critical (9)

- **C1 — live runtime bug.** `bridge.ts:515` invokes `'set_max_concurrency'`; only `set_max_concurrency_cmd` is registered (`coordinator.rs:632`, `lib.rs:125`). The Settings concurrency slider throws "command not found" today. **Fix:** change the bridge string to `'set_max_concurrency_cmd'`. [Lane W-A]
- **C2 — unvalidated sidecar boundary.** `apps/sidecar/src/index.ts:99` `JSON.parse(line) as SurfaceCommand` — bypasses `SurfaceCommandSchema`. **Fix:** `SurfaceCommandSchema.safeParse(JSON.parse(line))`, log+skip on failure (same as bad-JSON path). [Lane T]
- **C3 — web bypasses the authoritative contract and has drifted.** `apps/web` doesn't depend on `@nightcore/contracts`; `bridge.ts` re-declares `NcEvent` + every serde struct by hand. Drifted: engine emits `task-updated` (`sdk-adapter.ts:205`, in `NightcoreEventSchema` `events.ts:174`), Rust forwards verbatim (`sidecar.rs:162`), but `NcEvent` (`bridge.ts:163`) has no such variant → subagent steps silently dropped. All `invoke<T>()` + `readTranscript` (`bridge.ts:392`) cast with no runtime validation; the `isX` narrowers (`bridge.ts:627`) check 2–3 fields then trust the rest. **Fix:** add `@nightcore/contracts` dep to web; import the zod-inferred types instead of redeclaring `NcEvent`; add the `task-updated` member; `safeParse` transcript + event payloads. [Lane W-A]
- **C4 — no React error boundary.** A render throw blanks the whole WKWebView. **Fix:** top-level `ErrorBoundary` (componentDidCatch/getDerivedStateFromError) wrapping `<AppShell/>` in `App.tsx`, fallback with reload. [Lane W-A]
- **C5 — no user-facing error channel.** ~25 `invoke` rejections (run/cancel/merge/commit/approve/reject/delete/move/respondPermission/settings/project ops) dead-end in `console.error`. **Fix:** minimal Toast/error context primitive; route the catches through it. [Lane W-A]
- **C6 — streaming-transcript render storm.** Every `assistant-delta` replaces the board-wide `streams` object (`AppShell.hooks.ts:319`) → all columns + every unmemoized `TaskCard` re-render; `TaskDetail.tsx:308` re-`marked.parse`+`DOMPurify` the entire growing turn (O(n²)); `session-stream.ts:50` reallocates the timeline array per delta. **Fix:** `React.memo` `TaskCard`/`Column`; isolate the per-task live stream from board-wide state; render the open streaming entry as plain text until the turn closes; stable per-entry ids (not index keys). [Lane W-B]
- **C7 — `TaskStore::mutate` non-atomic read-modify-write.** `store.rs:142` get-clones-and-drops-lock then upsert re-locks; the spawned sidecar reader and Tauri handlers race the same task id, each persisting a full record and clobbering the other's fields. **Fix:** hold the `tasks` mutex across the whole read-modify-write; add a precondition variant so the status check + write share one lock acquisition (also fixes concurrency #2). [Lane R]
- **C8 — `delete_task` orphans worktree + branch.** `task.rs:440` removes task JSON + transcript but leaves the worktree dir and `nc/<id>` branch. **Fix:** mirror `merge.rs:116` cleanup when the task has a branch/worktree. [Lane R]
- **C9 — session-id counter not seeded.** `session-manager.ts:54` starts at 1 each launch; `SessionStore` collapses by id last-write-wins (`storage/index.ts:53`) → new sessions overwrite prior persisted records. **Fix:** seed `createMonotonicCounter(max(existing id)+1)` from the store. [Lane T]

## Suggested

### Contract / type-safety
- shallow event guards (`isTask` etc.) → `safeParse` against contracts schemas. [W-A, folds into C3]
- `session-status` typed `string` vs `SessionStatusSchema` enum (`bridge.ts:188`). [W-A]
- ESLint enables only `recommended` — `explicit-module-boundary-types`/`no-unsafe-*` off (`eslint.config.mjs:57`). **Enable `explicit-module-boundary-types` ONLY if it surfaces ≤ a handful of errors; do not enable `no-unsafe-*` (cascading). Defer with a note if it explodes.** [Lane T]
- `nightcoreTools: Array<SdkMcpToolDefinition<any>>` (`tools/index.ts:40`) → `ZodTypeAny`/`unknown` if it compiles. [Lane T]

### Concurrency (Rust unless noted) [Lane R]
- #2 status check-then-act split from `mutate` → precondition closure (fold into C7).
- #3 worktree allocate TOCTOU (`coordinator.rs:445`/`resolve_worktree`) → per-project async mutex around is_clean→allocate, or treat transient `worktree add` failure as retryable.
- #5 correlation FIFO mis-bind on reorder/crash (`provider.rs:212`) → evict the pending entry on terminal/abort. (The deeper "echo a correlation token on `session-started`" fix spans engine+contracts → note as deferred cross-lane.)
- #6 optimistic move stomps concurrent `nc:task` (`AppShell.hooks.ts:676`) → reconcile by `updatedAt`/skip optimistic for in-flight. [W-A]
- #7 `useBlockedIds`/`useWorktrees` refetch every event, no AbortController (`AppShell.hooks.ts:360,389`) → AbortController/request-id guard. [W-A]
- #11 sidecar crash: reader just `break`s (`sidecar.rs:82`) → drop stdin handle, drain correlation, fail/abort leased runs so the loop re-dispatches.
