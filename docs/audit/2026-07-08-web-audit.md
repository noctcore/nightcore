## Cross-Cutting Findings (highest leverage)
- `runStatusOf(...)` dead export duplicated in 4 features (harness/insight/scorecard/prreview constants). Actual usage is from `@/lib/scan-run`. Remove all copies (safe dead code).
- Grandfathered file-size ratchet entries (web-file-size-ratchet + carve-outs in eslint.config) across multiple large files: AppShell, bridge/events, IssueTriageView.hooks, TaskDetail, ProviderConfigPanel, HarnessView, InsightView.hooks, prreview-lifecycle, etc. Only shrink + update baseline together.
- Direct `lib/generated/*` imports in prreview tests (contract violation; should go through `@/lib/bridge`).
- Missing test harness for the 18 lint-meta rules (high risk; all enforcement currently integration-only via live tree).
- 4 lint-meta rules undocumented in root AGENTS.md (`no-cloned-component-folders`, `scan-family-parity`, `agent-contract-parity`, `codegen-drift`).
- UI primitives: a few folder composites missing `.types.ts` (Modal, Toolbar, Badge shape decision) while flat primitives correctly stay flat.
- New Project flow: `model`/`concurrency` collected in dialog + MODELS labels but dropped by caller (dead data path + id vs label mismatch).
