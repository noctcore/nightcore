# Audit Findings тАФ web-ui

**Date:** 2026-07-08
**Agent:** kirei-refactor (audit slice)
**Scope:** apps/web/src/components/ui (all files and subdirectories)

## Summary

Audited the UI primitives layer (~236 source files across 50+ components). This slice is the dependency-graph leaf for rendering: primitives accept data via props and MUST NOT import feature folders or `@tauri-apps/*`. All findings respect that boundary.

**Overall health:** High. No `any` types, no console.log in production paths, no feature/tauri imports, no upward layer violations. Most folder composites follow the full sibling set pattern.

**Key themes:**
- Three barrel exports (`STATIC_MODEL_CATALOG_DATA`, `useModelCatalog`, `PROVIDER_GLYPHS`) are re-exported publicly but only consumed internally via relative paths.
- Two folder composites (Modal, Toolbar) define their `*Props` in the `.tsx` rather than a sibling `.types.ts`, violating the documented naming contract.
- Several flat-file primitives correctly keep props inline; a few folder components that should have graduated to `.types.ts` have not.
